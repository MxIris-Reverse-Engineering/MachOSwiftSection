# 0.13.0

Headline work since `0.12.0`:

1. **`SwiftLayout` resolves generic types without arguments** — phases 7–9 of the static layout engine: associated-type fields, extended existentials, parent-chain generic arguments, `@objc` protocol existentials, class-bound generic parameters, parameter metatypes, and same-type / same-value pinned parameters now lay out exactly with **no specialization**. Field degradation on the 5-framework non-generic survey dropped to **0%**.
2. **Exact extra-inhabitant modeling** — every leaf XI count now matches the live runtime value witness table (verified size / stride / alignment / XI / bitwise-takability for every fixture type), fixing a family of single-payload enum size and trailing-field offset bugs.
3. **Runtime enum-case projection + layout audit** — enum-layout comments now show each case's exact fixed bytes when the metadata is live in-process, and the calculator was audited line-by-line against the Swift runtime sources with five fidelity fixes.
4. **`OutputTransformer`** — a new bottom-of-graph module hosting the token-template comment renderers (field offset, vtable offset, member address, type layout, enum layout) with five enum-layout presets, wired through the CLI's `--enum-layout-style`.

## Highlights

### `SwiftLayout` — unspecialized generic layout (phases 7–9)

- **Phase 7**: associated-type fields of concrete instantiations (`C.Index` via `__swift5_assocty` witnesses), constrained/extended existentials (`any Boxed<Int>`), and nested types of specialized generic parents.
- **Phase 8**: generic arguments are collected **per level along the nominal parent chain**, so `Environment<Bool>.Content`-style nested types bind the parent's arguments; Swift-declared `@objc` protocol existentials resolve through a `__objc_protolist` fallback. These two fixes took the non-generic survey's field degradation from ~4% to **0%**.
- **Phase 9**: what a generic descriptor's requirement signature pins without any argument is now mined in one pass — **class-bound parameters** (`Element: AnyObject` / `: SomeClass` / `: SomeClassBoundProtocol`) lay out as a single object reference, **concrete same-type pins** from constrained extensions (`Value == Date`) become genuine substitutions, **same-value pins** (`count == 5`) bind like value arguments, and a parameter metatype field (`T.Type`) is always a thick metadata pointer. A generic-type survey (2963 types, 7255 fields) shows ~52% of generic struct/class fields resolving with no argument at all.
- **Value generics + parameter packs**: `ValueGenericBuffer<5>`, `InlineArray<3, Int64>`, and flat-pack instantiations resolve both as fields and as top-level requests (`Builtin.FixedArray` ported from `swift_getFixedArrayTypeMetadata`).

### Exact extra-inhabitant counts, verified against the runtime

- Value aggregates derive XI = max over fields (`swift_initStructMetadata` semantics) — previously struct XI defaulted to 0, mis-sizing any single-payload enum over a struct payload (the `SwiftUI.Text.Style` cascade).
- Frozen `String`/`Character` carry the `_StringObject` discriminator count (0x7FFFFFFF); raw pointers reserve only null (XI 1); thick functions carry the saturated count on the function-pointer word (fixing `Optional<() -> Void>` from 17 bytes back to 16); `weak` storage is XI 0 and not bitwise-takable; `unowned` is exactly XI 1.
- Multi-payload enums compute exact XI per strategy (IRGen `getFixedExtraInhabitantCount` for spare-bits/hybrid, the runtime tagged formula), so `Optional<MPE>` wrapping stays exact even on the structural fallback — the official offline implementation (RemoteInspection) never derives spare-bits XI structurally.
- `WholeTypeLayoutVsRuntimeTests` now asserts the full quintuple — size / stride / alignment / **extraInhabitantCount** / **isBitwiseTakable** — against the runtime VWT for every fixture struct and enum.

### Runtime enum-case projection + enum-layout audit

- `RuntimeEnumCaseProjector` resolves each case's fixed bytes **exactly** when the enum's metadata is live: dual-baseline (`0x00`/`0xFF`) `destructiveInjectEnumTag` injection diffs out the pattern, `getEnumTag` round-trips guard empty cases, arm64e ptrauth degrades to `nil`. `Text.Style.LineStyle`-style comments now read `bytes[0x8..<0x10] = 0x1` instead of a placeholder; the offline path renders an honest "not resolved offline" note instead of fabricating patterns.
- The calculator was audited against `EnumImpl.h` / `Enum.cpp` / `GenEnum.cpp` / `TypeLowering.cpp` with five fixes: indirect single-payload enums are XI layouts over the box pointer (not overflow), enum sizes are cross-checked against the VWT, spare-bits payload cases carry bit-level `fixedBitMasks` (no whole-byte over-claiming), empty cases record the whole payload area, and no-payload XI is capped correctly.
- The default enum-layout rendering is now the `detailed` style.

### `OutputTransformer` — token-template comment modules

- New dependency-free module hosting the `Transformer` namespace RuntimeViewer's settings UI edits: `SwiftFieldOffset`, `SwiftVTableOffset`, `SwiftMemberAddress`, `SwiftTypeLayout`, `SwiftEnumLayout` (three template levels, `${token}` placeholders).
- Five enum-layout presets — `detailed` (default, test-pinned identical to `EnumCaseProjection.description`), `explained` (bit ranges in plain words), `standard`, `inline`, `compact` — selectable via `swift-section dump --enum-layout-style <preset>`.
- Wiring goes through `applyTransformers(_:)` on the render/print configurations; emission sites unchanged. RuntimeViewerCore reduces to an `@_exported` shim (ObjC-side modules stay there for now).

### Other notable surface

- **SE-0452 value-generic integer nodes** now render in interfaces — `SwiftPrinting` previously dropped them, printing `ValueGenericBuffer<>` / dangling `where A ==`.
- **`TypeLayoutInfo` → `StaticTypeLayout`** rename in `SwiftLayout`'s API.

## Bug Fixes

- **`SwiftLayout`** — value-aggregate XI derivation (struct payloads were mis-sized); `String` XI corrected from 1; raw-pointer XI corrected from the 0x1000 approximation (mis-sized two-empty-case enums); thick-function XI corrected from 0 (a real trailing-field offset bug); single-payload generic enums read the correct parameter instead of blindly the first type argument.
- **`SwiftInspection`** — the five enum-layout audit fixes above; single-payload projections apply exact case patterns from live value witnesses.
- **`SwiftPrinting`** — SE-0452 integer nodes no longer dropped.
